# 正本ドキュメント・ADR 自動探索ルール

Issue / Discussion 作成時に、関連する正本ドキュメント・ADR・既存 Issue を自動探索し、本文の `## 関連参照` セクションとして組み込むための共通ロジックを定義する。

## 適用スキル

| スキル | 適用ステップ | 出力先 |
|--------|------------|------|
| `create-item-flow` | ステップ 1c | Issue 本文（`## 目的` 直後・`## 概要` 直前） |
| `requirements-flow` | ステップ 2 拡張 | Discussion 本文（ADR / 仕様の各テンプレート位置に応じる） |

## 設計原則

1. **YAGNI 開始**: シンプル案（タイトル名詞句をそのまま使う）で開始。ヒット率が低ければ高度化を検討
2. **フェイルセーフ**: 探索結果が 0 件でもエラーにならず、セクションを省略してそのまま次へ進む
3. **読者ファースト**: 発見した参照は本文の冒頭近くに配置し、コンテキスト不足の reviewer が前提知識を補えるようにする
4. **冗長性より精度**: 5 件を超える Issue / ADR がヒットした場合は厳しめにフィルタする（ノイズ防止）

## 探索対象とコマンド

| カテゴリ | 検索方法 | 上位件数 | 出力先サブセクション |
|---------|---------|---------|------------------|
| 正本ドキュメント | `grep -l "<keyword>" -r docs/guide/ docs/specs/ CLAUDE.md` で該当ファイルを発見 → 関連箇所を Read で抜粋 | 5 件まで（重要箇所のみ） | `### 正本ドキュメント` |
| ADR (Accepted) | `shirokuma-flow discussion adr list` → タイトル / 本文をキーワードで照合 | 3-5 件 | `### 関連 ADR（Accepted）` |
| 既存 Issue / PR / Discussion | `shirokuma-flow issue search "<keyword>" --limit 5` | 5 件まで | `### 関連既存 Issue / PR` |

## キーワード生成

### シンプル案（デフォルト）

タイトルから以下の優先順で抽出:

1. **タイトル全文を 1 クエリとして使用**（`issue search` は全文検索のため）
2. タイトル中の **名詞句**（複合語含む）を分割して個別クエリにする

#### 日本語タイトルのフォールバック分割

形態素解析は使わず、以下のルールで単純分割する:

- **区切り文字**: スペース・記号・カッコ（`（）「」『』 ・,.:;`）で分割
- **長さフィルタ**: 2 文字以上の連続文字列を候補とする（1 文字は除外）
- **ストップ語の簡易除外**: 「する」「した」「など」「こと」「もの」のみ除外（過剰な除外はしない）

#### 例

| 入力タイトル | 分割結果（候補キーワード） |
|------------|--------------------------|
| `feat(skill): create-item-flow に正本ドキュメント・ADR 自動探索ステップを追加` | `create-item-flow`, `正本ドキュメント`, `ADR`, `自動探索`, `ステップ`, `追加` |
| `fix: Review 遷移仕様乖離を解消` | `Review`, `遷移仕様`, `乖離`, `解消` |
| `chore: 依存パッケージを更新` | `依存パッケージ`, `更新` |

### 悪化判定シグナル（仕様の見直しトリガー）

以下のいずれかが発生した場合、シンプル案では不十分と判断し、ルール再考を提案する:

- **3 連続で 0 件ヒット**: 3 回連続で全カテゴリが 0 件 → ストップ語追加 / 形態素解析導入を検討
- **ヒット率の偏り**: 全カテゴリで毎回 10 件以上ヒット（ノイズ過多）→ tf-idf 風の重み付け導入を検討
- **同義語の取り逃し**: 「Issue」と「課題」、「Plan」と「計画」など同義語による取りこぼしが目立つ → 同義語辞書を補助 reference に追加検討

これらのシグナルは Issue コメントまたは Evolution Issue として記録する（蓄積後にまとめて評価）。

## 本文への組み込み

### 配置位置（スキル別）

| スキル / 種別 | 配置位置 |
|------------|--------|
| `create-item-flow`（Issue / Discussion） | `## 目的` 直後・`## 概要` 直前 |
| `requirements-flow`（ADR） | `## Status` 直後・`## Context` 直前 |
| `requirements-flow`（仕様 Discussion） | `## 目的` 直後・`## 概要` 直前（create-item-flow と同じ） |

### 共通テンプレート

```markdown
## 関連参照

> **このセクションの趣旨**: 関連する正本・ADR・過去 Issue を冒頭に列挙する。読者はここから前提知識を補える。

### 正本ドキュメント
| ドキュメント | 関連箇所 | 本 Issue / 本 ADR / 本仕様との関係 |
|------------|---------|--------------------------------|
| ... | ... | ... |

### 関連 ADR（Accepted）
| ADR | 番号 | 関係性 |
|-----|------|--------|
| ... | ... | ... |

### 関連既存 Issue / PR / Discussion
| 番号 | 状態 | 関係性 |
|------|------|--------|
| ... | ... | ... |
```

### サブセクション省略ルール

| 状態 | 挙動 |
|------|------|
| 1 カテゴリのみ 0 件 | 該当サブセクションのみ省略 |
| 全カテゴリ 0 件 | `## 関連参照` セクション全体を省略 |

セクション全体が省略された場合、本文は通常のテンプレート構造になる。

## フィルタ・上限

### 件数上限

- **正本ドキュメント**: 5 ファイルまで（多すぎる場合は最も関連が高いものを選定）
- **ADR**: 3-5 件まで（Accepted 状態のみ。Deprecated / Superseded は除外）
- **Issue / PR / Discussion**: 5 件まで（Open + 直近半年の MERGED/CLOSED 上位）

### 関連箇所の抜粋ルール

正本ドキュメントの「関連箇所」列には行番号 + 短い要約を記載する。長文の引用は避ける（本文が肥大化するため）。

```
| `docs/guide/workflows/lifecycle-overview.md` | "Issue 作成" シーケンス L49-58 | create-item-flow → analyze-issue → Status 遷移の正本フロー |
```

## `analyze-issue requirements` との責務境界（create-item-flow 適用時）

事前探索（このルール）と `analyze-issue requirements` のプロジェクト要件整合性チェック（事後検証）は **時系列と目的が異なる相補的役割** を持つ。両者を統合せず、各々の役割を維持すること。

| 観点 | 事前探索（このルール） | `analyze-issue requirements` |
|------|------------------|----------------------------|
| 実行タイミング | アイテム **作成前** | アイテム **作成後** |
| 主目的 | 関連参照を網羅的に本文へ埋め込む | ADR 整合性を検証し PASS / NEEDS_REVISION 判定 |
| 出力先 | 本文の `## 関連参照` セクション | Issue コメント |
| 失敗時の挙動 | 0 件ならセクション省略（フェイルセーフ） | NEEDS_REVISION でループ修正 |

## requirements-flow 既存ステップ 2 との統合

`requirements-flow` SKILL.md ステップ 2「既存 ADR・仕様との重複・矛盾を確認する」は既に類似検索を実施している。本ルールはステップ 2 を **拡張** する形で適用する:

1. ステップ 2 の検索結果を Discussion 本文の `## 関連参照` セクションに整形して埋め込む
2. 検索結果が 0 件のカテゴリは省略
3. 全カテゴリ 0 件の場合は `## 関連参照` セクション全体を省略

これにより「重複・矛盾チェック」と「reviewer コンテキスト補完」の両方が同じ検索結果から得られる。
