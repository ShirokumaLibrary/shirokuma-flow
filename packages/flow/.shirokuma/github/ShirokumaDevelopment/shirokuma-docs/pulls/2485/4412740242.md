### Critical #1 詳細

`packages/flow/src/commands/issue/update/index.ts` で `cmdItemUpdate` 入口に以下を追加:

```ts
// [body-file] positional をファイル内容に解決する。
// インライン文字列を本文として受理してしまう dual-path をここで構造的に封鎖する (#2485 Critical-1)。
if (!resolveBodyFileOption(options as Record<string, unknown>)) return 1;
```

これにより `updateIssue/updateDiscussion/updateComment` 内の `existsSync(options.bodyFile)` 分岐は不要となり削除。`shirokuma-flow issue update 42 "本文文字列"` のようなインライン受理は「ファイルが見つかりません」エラーで構造的に拒否されます。Issue で明示されていた stdin (`-`) サポートも `resolveBodyFileOption` 経由で復活。
