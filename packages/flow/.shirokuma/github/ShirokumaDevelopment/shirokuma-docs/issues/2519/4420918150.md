## 作業サマリー

`shirokuma-docs lint structure -f json` で Commander の親子オプション解決により `-f json` がサイレント破棄され、info ログが stdout を汚染する問題を修正した。

### 修正内容

`packages/flow/src/commands/lint/index.ts` の `mergeOpts` ヘルパーを拡張。子コマンドが親と同じ option を再宣言した場合、Commander v14 は CLI トークンを親側で parse する一方、子の `localOpts` は default 値のまま残る。素朴な `{ ...parentOpts, ...localOpts }` merge では子の default が親の cli 値を上書きしていた。

修正後は `command.getOptionValueSource(key)` で child / parent の source を判定し、`childSource === "default"` かつ `parentSource !== "default"` のときに parent 値を優先する。

### 受け入れ基準

- [x] `shirokuma-docs lint structure -f json | jq .` がエラーなく動作する
- [x] `lint all` 経由で structure が JSON 出力する（既存挙動を維持）
- [x] 他フォーマット (`terminal` / `yaml`) の動作は変わらない

### テスト

`packages/flow/__tests__/commands/lint-merge-opts.test.ts` に 4 ケース追加（shadowing 回帰防止）:
- `lint structure -f json` で子 default に上書きされず `"json"` が伝播
- 引数なし時は子 default `"yaml"` が優先される
- 子に明示的に `-f terminal` を指定すれば伝播
- 親側 `lint -f json structure` でも子に伝播

### フォローアップ候補

simplify レビューで `packages/flow/src/commands/skill/index.ts` 等に同名 `mergeOpts` のローカル定義（修正前の古いバージョン）と、他コマンド `commands/*/index.ts` 配下で `parent?.opts()` の inline 展開が 80+ 箇所あることが判明。同じ shadowing バグを抱える可能性があり、共通 util への昇格は別 Issue で検討する価値がある。
