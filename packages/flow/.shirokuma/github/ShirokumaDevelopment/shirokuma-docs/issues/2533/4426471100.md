## sub3 スコープ追加（PR #2537 レビュー指摘より）

PR #2537（sub1）のコードレビューで以下の **High / Critical 級指摘** が sub3 のスコープに追加されました。本 Issue 着手時に追加対応してください。

### Critical: `Approved` LEGACY 三者不整合の解消

sub1 で `normalizeLegacyStatus("Approved") → Done` に変更したが、`integrity/index.ts:874-891` の `--fix` が依然 `Approved → Review` のまま。三者不整合（`migrationTarget` メタデータ vs description vs `--fix` 実動作）を解消する必要がある。

**対象ファイル:**
- `packages/flow/src/commands/items/integrity/index.ts:874-891`:
  - `--fix` 動作を `STATUS_VALUES.REVIEW` → `STATUS_VALUES.DONE` に変更
  - `fixes.action` を `"approved-to-review"` → `"approved-to-done"` にリネーム
  - `logger.success("Reverted #${item.number} status: Approved → Review")` → `"Migrated #${item.number} status: Approved → Done"`
- `packages/flow/src/commands/items/integrity/classify.ts:550-555`:
  - 通常 Issue が Approved の場合の description（`Approved は plan/design Issue 専用です（--fix で Review に戻します）`）を `Done に統合します` に変更

### High: `pr/merge.ts:425` の `isBacklogEquivalent` 使用箇所を細分化判定に置換

sub1 で `isBacklogEquivalent` が `Backlog` (未調査) と `ToDo` (着手準備完了) の両方を含むよう拡張された。`pr/merge.ts:425` の「次の `Backlog` 計画」を選ぶフィルタは `isReadyForImplementation` (ToDo 専用) または `isInvestigationPending` (Backlog 専用) に置換する判断が必要。

「実装着手可能 = ToDo」が新モデルの意味なので、推奨は `isReadyForImplementation` への置換。メッセージ `次の計画 #N が Backlog です` も `次の計画 #N が ToDo です（着手準備完了）` 等に修正。

### High: `parent-status.ts:389` の `isBacklogEquivalent` 使用箇所の意味変更対応

`allBacklog = subIssueStatuses.every((s) => isBacklogEquivalent(s))` の派生ロジックを 6 値モデル対応に書き換え:

- 全サブが `Backlog` (未調査) → 親も `Backlog`
- 全サブが `ToDo` (着手準備完了) → 親 `ToDo` (現状の挙動継続)
- Backlog と ToDo の混在 → 「より低い段階」の `Backlog` を採用

`isInvestigationPending` (Backlog 専用) と `isReadyForImplementation` (ToDo 専用) を組み合わせて分岐実装する。

### High: 旧 `STATUS_TRANSITIONS[Review]` から `STATUS_VALUES.TODO` を削除

sub3 で旧 API (`STATUS_TRANSITIONS` / `validateStatusTransition(from, to)`) を削除する際に、新モデルで禁止された `Review → ToDo` 遷移を旧定義からも削除する。これは `approve` 意味変更（`Review → Done`）と整合させる必要がある。

呼び出し側全箇所:
- `packages/flow/src/commands/issue/close/index.ts:75`
- `packages/flow/src/commands/issue/update/index.ts:409`
- `packages/flow/src/commands/issue/reopen/index.ts:43`
- `packages/flow/src/commands/status/transition/index.ts:250`
- `packages/flow/src/commands/status/allowed.ts:37,70`
- `packages/flow/src/commands/status/get.ts:41`
- `packages/flow/src/utils/project-fields.ts:387`

これらを新 API (`validateTransition(itemType, from, to, opts)`) に切り替え + 旧 `STATUS_TRANSITIONS` / 旧 `validateStatusTransition` 削除。

### High (関連): `INCOMPLETE_SUB_ISSUE_STATUSES` の `APPROVED_LEGACY` 残置検討

sub3 で Approved を持つ既存 Issue を migrate-v3-018（または `--migrate-v3-022`）で Done にマイグレーションし、その後 `APPROVED_LEGACY` を `INCOMPLETE_SUB_ISSUE_STATUSES` から削除する戦略を sub3 / sub6 で検討。
