## 実装完了サマリー

PR #2487 で `discussion add` の本文受け渡し経路を **positional file argument に統一**しました。`issue add <file>` / `issue comment <number> <file>` / `pr create [body-file]` と CLI シグネチャの対称性が回復し、PR #2485 で達成した「インライン body の物理的封鎖」を `discussion add` にも拡張しています。

**PR:** #2487 (`feat/2486-positional-discussion-add` → `develop`)

### 主な変更

| 変更カテゴリ | 内容 |
|------------|------|
| Commander signature | `discussion add --file <file>` → `discussion add <file>` (required positional, `-` で stdin) |
| `cmdAddDiscussion` | 第1引数に `file` を追加、`AddDiscussionOptions` から `file`/`body` フィールドを削除 |
| 共有ヘルパ拡張 | `readFileWithFrontmatter` に `-` sentinel 経由の stdin 読み込みを追加（hybrid: 通常パスは `readFile`、`-` のみ `readBodyFile`） |
| 新規テスト | `discussion-positional-body-file.test.ts` で `--file` rejection / signature / positional 伝達 / stdin 伝達を検証 |
| ドキュメント一括更新 | 46 ファイル: `.shirokuma/rules/`、`docs/specs/`、`docs/guide/`、`plugin/shirokuma-skills-{en,ja}/`、`plugin/shirokuma-{nextjs,infra}-{en,ja}/`、`agents/` 内の `discussion add --file` 言及を positional 形式に統一 |

### /simplify レビュー改善

simplify による 3 軸並列レビュー（reuse / quality / efficiency）で HIGH 共通指摘を検出し、追加修正を行いました:

- **HIGH (reuse + quality)**: 初版は `cmdAddDiscussion` が `readFileWithFrontmatter` を inline 化していたため、共有ヘルパを stdin 対応に拡張して再利用に戻した。これにより `delete meta["body"]` の挙動ドリフトと `parsed.hasFrontmatter` の二重チェックが同時解消
- **MEDIUM (quality)**: コード内 `#2486` task ref をヘッダ・JSDoc から除去
- **MEDIUM (test)**: `--file` 拒否テストの過広な assertion `errorMessage.includes("error")` を削除し、`unknown option` のみで判定
- **EFFICIENCY**: clean（hybrid 路 `readFile` のまま、TOCTOU は pre-existing で本 PR 範囲外）

### セキュリティレビュー
