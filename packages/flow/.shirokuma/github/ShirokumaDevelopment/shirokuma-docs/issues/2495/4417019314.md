### 実装内容

**A. `projects` コマンド完全削除**

- `src/index.ts` から `createProjectsCommand` の import / 登録を削除
- `src/commands/projects/` 配下の実装 12 ファイルを `src/commands/project/` に移動し、`project/index.ts` の動的 import 10 件を `./list.js` 等の同階層パスに書き換え
- `src/commands/projects/index.ts`（コマンド登録ファイル）を削除
- `src/commands/items/integrity/index.ts` のインポートを `../../project/setup-metrics.js` に切り替え
- `src/utils/project-fields.ts` の警告メッセージを `project setup-metrics` に変更
- テスト削除: `__tests__/commands/projects/`（3 ファイル）+ `projects.test.ts`
- 関連テスト 4 ファイルの `jest.unstable_mockModule` パスを更新

**B. `update-skills` を `update` に統合**

- `update` コマンドに `--with-rules`, `--sync` オプションを追加（`--sync` はデフォルト true、`--no-sync` で解除可）
- `update-skills` コマンド登録ブロックを削除（実装ファイル `update-skills.ts` は存続）
- `__tests__/commands/update-skills.test.ts` を `update` コマンドに対応させ、`--with-rules`/`--sync` 受け入れテストを追加

**C. `generate github-data` を `sync-github` トップレベル化**

- `src/commands/generate/github-data.ts` を `src/commands/sync-github/index.ts` に移植
- `src/commands/generate/` ディレクトリ全体を削除
- `src/index.ts` で `sync-github` コマンドを動的 import 付きインライン登録
- 参照置換: `cli-quick-reference.md`, `contributing.md`, `CLAUDE.md`

**ドキュメント参照置換**

- 10 ファイル: `CLAUDE.md`, `cli-quick-reference.md`, `deprecated.md`, `plugins.md`, `contributing.md`, `config-reference.md`, `github-commands.md`, `plugin-cache.md`（EN/JA）, `switch-plugin.sh`, `sync-versions.sh`

### finalize-changes での改善

`--sync` デフォルト処理の冗長な三項演算子 `options.sync !== false ? true : options.sync` を単純比較 `options.sync !== false` に簡略化した（コミット `712e353`）。

### セキュリティレビュー結果

脆弱性なし。新規 `exec`/`spawn`/`eval` シンク、GraphQL クエリ改変、ユーザー入力経由のシェル実行などはすべて該当なし。

**注記（別 Issue 候補）:** 新 `sync-github` インライン登録に、旧 `generate` コマンドが持っていた `validateProjectPath` preAction フックが含まれていない。同一ユーザー実行で権限境界はないためセキュリティリスクはないが、コンシステンシー上の差異として追跡余地あり。
