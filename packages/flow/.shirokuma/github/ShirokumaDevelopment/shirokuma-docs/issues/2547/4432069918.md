### 個別検証

#### Critical 対応の検証（`rollback/index.ts:397` への明示フック追加）

実装確認:
- `rollback/index.ts:394-398` の現状は `getIssueId → unparentFromOwnParent → closeIssueById(issueId, "NOT_PLANNED")` の順で `syncChildCloseOnParentClose` を経由しない（前回指摘どおり）。
- 計画では「`closeIssueById` 呼び出し直前に `syncChildCloseOnParentClose(owner, repo, number, logger, "NOT_PLANNED")` を追加」と方針 B を採用。
- `syncChildCloseOnParentClose` シグネチャは `(owner, repo, parentIssueNumber, logger, parentStateReason)` で第 5 引数に `NOT_PLANNED` を渡せる（`parent-status.ts:764-770`）。整合的。
- 変更対象ファイルに `packages/flow/src/commands/issue/rollback/index.ts` が追加されており、タスク分解にもフック追加タスクが入っている。

#### `pr merge` 経由スコープ外宣言の妥当性

実装確認:
- `pr/merge.ts:243` の `syncChildCloseOnParentClose(owner, repo, num, logger)` は PR にリンクされた実作業 Issue (`num`) の **直下子** に呼ばれる。実作業 Issue は通常子を持たないため空連鎖。
- `closeDoneParent`（`parent-status.ts:461-490`）は `closeIssueById` で親を直接 close するのみで `syncChildCloseOnParentClose` を呼ばない。よって「PR マージ → 親 Done → 兄弟子 cascade」ルートは現状未実装。
- スコープ外宣言は実装現状と一致しており妥当。
- 計画タスクで「`pr/merge.ts` の `syncChildCloseOnParentClose` 呼び出し近傍にスコープ外コメント追記」も含まれており、将来の誤実装防止策まで配慮されている。
