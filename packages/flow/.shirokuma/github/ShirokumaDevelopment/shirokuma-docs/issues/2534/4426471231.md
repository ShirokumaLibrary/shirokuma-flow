## sub4 スコープ追加（PR #2537 レビュー指摘より）

PR #2537（sub1）のコードレビューで以下の **Medium 級指摘** が sub4 のスコープに追加されました。**本 Issue 着手時の最優先タスク**として実施してください。

### Medium (最優先): `add.legacy.ts.disabled` の status 非依存 28 テストの即時復活

sub1 で `__tests__/commands/issue/add.test.ts` を `add.legacy.ts.disabled` にリネームしましたが、このファイルには status 検証以外の以下のテストが多数含まれています:

- `cmdAddComment` 6 テスト（Issue/PR/Discussion 自動判別）
- `cmdAddIssue` の labelRules / IssueType 解決 / assignees / cache 系 約 17 テスト
- `cmdAddDiscussion` 4 テスト

これらは status 6 値化と無関係な機能カバレッジであり、sub4 着手時に即座に復活させる必要があります。

**手順:**

1. `add.legacy.ts.disabled` のうち status 検証 5 テスト（`status が Review`, `status が In progress`, `status が Pending`, `status 省略 + getDefaultStatus()`, `バリデーションエラーになることを確認`）のみを別ファイル `add.status.legacy.ts.disabled` に分離
2. 残り 28 テストを `add.test.ts` に戻す（テスト内の status 期待値を `Backlog` に更新）
3. `getDefaultStatus()` モックの戻り値を `"In progress"` → `"Backlog"` に変更
4. `__tests__/utils/status-workflow.legacy.ts.disabled` も同様に評価し、status 非依存テスト（あれば）を復活

### 補足: status-workflow.test.ts の 6 値版作成

sub1 で `status-workflow.test.ts` を `.legacy.ts.disabled` にリネームしました。sub4 では以下を新規作成してください:

- `__tests__/utils/status-workflow.test.ts` (新規 6 値版): 4 テーブル × 6 値 × Forward/Rollback の網羅テスト + `getAllowedTransitions` / `validateTransition(itemType, ...)` シグネチャテスト
- 必要に応じて `status-workflow-6values.test.ts` として独立ファイルも検討（sub1 で削除した `status-workflow-5values.test.ts` の対応形）
